01

Document Structure

The skeleton of every HTML page — the required tags that frame all content and tell browsers how to read the file.

<!DOCTYPE html> Structure Declares the document as HTML5 ▼
The DOCTYPE declaration must be the very first line of every HTML document. It tells the browser to render the page in standards mode (not quirks mode). Without it, older browsers may misinterpret your CSS and layout. It is not a tag — it is an instruction to the browser.
Syntax
<!-- Always the very first line of your HTML file -->
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Page</title>
  </head>
  <body>
    <p>Hello, world!</p>
  </body>
</html>
💡 DOCTYPE is not case-sensitive, but <!DOCTYPE html> (lowercase) is the modern convention.
<html> Structure Root element — wraps all page content ▼
The <html> element is the root of the HTML document. Every other element must be inside it. The lang attribute is critical for accessibility — screen readers and search engines use it to detect the page language.
Key Attributes
langLanguage code: "en", "fr", "th", "zh". Essential for accessibility and SEO.
dir"ltr" (left-to-right, default) or "rtl" (right-to-left, for Arabic, Hebrew).
Example
<html lang="en"> <!-- English -->
<html lang="th"> <!-- Thai -->
<html lang="ar" dir="rtl"> <!-- Arabic, right-to-left -->
<head> Meta Container for metadata — not displayed on screen ▼
The <head> section contains metadata about the document — information the browser and search engines need, but users don't see directly. It holds the title, charset declaration, CSS links, scripts, and meta tags.
Full head example
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Page Title</title>
  <meta name="description" content="Page description for search engines">
  <link rel="stylesheet" href="styles.css">
  <link rel="icon" href="favicon.ico">
</head>
<body> Structure All visible page content lives here ▼
The <body> element contains everything that is rendered and visible on the web page — text, images, links, forms, videos, and all other content. There can only be one <body> per HTML document.
Example
<body>
  <header>...</header>
  <main>
    <h1>Welcome</h1>
    <p>All your content goes inside body.</p>
  </main>
  <footer>...</footer>
</body>
<title> Meta The page title shown in browser tab and search results ▼
The <title> defines the document title that appears in the browser tab, bookmarks, and search engine results (SERPs). It is one of the most important SEO elements. Keep it under 60 characters for best display in Google.
Example
<title>HTML Functions Book — Complete Reference</title>

<!-- Pattern: Page Name | Site Name -->
<title>Contact Us | MyWebsite</title>
💡 Keep titles under 60 characters. Include your main keyword near the beginning.
<div> Structure Generic block-level container with no semantic meaning ▼
The <div> is a generic block-level container. It creates a new "box" that takes up the full width. Use it for layout and grouping when no semantic element applies. Avoid div-itis — prefer semantic tags like <section>, <article>, <nav>.
Example
<!-- Layout wrapper -->
<div class="container">
  <div class="sidebar">...</div>
  <div class="content">...</div>
</div>

<!-- JavaScript target -->
<div id="app"></div>
⚠ Prefer semantic elements (<section>, <article>, <nav>) over <div> when content has meaning.
<span> Text Generic inline container with no semantic meaning ▼
The <span> is the inline version of <div>. It wraps inline content (text, images) without creating a line break. Use it to apply styles or JavaScript to part of a sentence without affecting layout.
Example
<p>Price: <span class="price">$29.99</span></p>

<p>Status: <span id="status" style="color:green">Online</span></p>
<!-- comment --> Structure Adds notes in HTML code that browsers ignore ▼
HTML comments let you add notes to your code that browsers do not render. They are visible in View Source and DevTools — never put passwords or secrets in comments! Used for documentation, temporarily disabling code, or leaving TODOs.
Example
<!-- This is a single-line comment -->

<!--
  Multi-line comment:
  TODO: Add mobile menu here
-->

<!-- <p>Temporarily disabled content</p> -->
⚠ Comments are visible in page source! Never store API keys, passwords, or private notes in HTML comments.
02

Semantic Layout

HTML5 semantic elements give meaning to page regions — improving accessibility, SEO, and code readability.

<header>SemanticIntroductory content: site logo, nav, page heading▼
The <header> represents introductory content. Used for the site header (containing logo, navigation) or as a header within an <article>/<section>. A page can have multiple headers — each section or article can have its own. Not the same as <head>!
Example
<header>
  <img src="logo.svg" alt="Logo">
  <nav>
    <a href="/">Home</a>
    <a href="/about">About</a>
  </nav>
</header>
<nav>SemanticA block of navigation links▼
The <nav> element represents a section of navigation links. Use it for primary menus, breadcrumbs, or tables of contents. Screen readers can skip navigation blocks using this landmark. Not every link group needs <nav> — use it only for major navigation.
Example
<nav aria-label="Main navigation">
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/blog">Blog</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>
<main>SemanticThe dominant/unique content of the page — only one per page▼
The <main> element contains the primary, unique content of the document. There should be only one <main> per page. It excludes repeated content like sidebars, navigation, headers, and footers. Screen readers use it as a "skip to main content" target.
Example
<body>
  <header>...</header>
  <main>
    <!-- The unique content of this page -->
    <h1>About Our Company</h1>
    <p>We make great products...</p>
  </main>
  <footer>...</footer>
</body>
<section>SemanticA thematic grouping of content with a heading▼
The <section> groups related content together — like chapters in a book. Each section should typically have a heading (h2, h3…). Use <section> when the content could be listed in a document outline. When in doubt between <div> and <section>: if it needs a heading, use <section>.
Example
<main>
  <section>
    <h2>Our Services</h2>
    <p>We offer web design, SEO, and hosting.</p>
  </section>
  <section>
    <h2>Our Team</h2>
    <p>Meet the people behind the work.</p>
  </section>
</main>
<article>SemanticSelf-contained, independently distributable content▼
An <article> is a self-contained piece of content that could stand alone or be reused elsewhere — blog posts, news articles, forum posts, user comments, product cards. Ask yourself: "Could I copy this to another website and it would still make sense?" If yes, use <article>.
Example
<article>
  <header>
    <h2>10 Tips for Better HTML</h2>
    <time datetime="2024-01-15">January 15, 2024</time>
  </header>
  <p>Good semantic HTML improves SEO...</p>
  <footer>By <address>Jane Smith</address></footer>
</article>
<aside>SemanticTangentially related content — sidebars, callouts▼
The <aside> represents content tangentially related to the main content — sidebars, pull quotes, ads, tip boxes, related links. It can be nested inside an <article> for article-specific sidebars, or be a top-level sidebar alongside <main>.
Example
<article>
  <p>The main article content...</p>
  <aside>
    <h3>Did you know?</h3>
    <p>HTML was created by Tim Berners-Lee in 1991.</p>
  </aside>
</article>
<footer>SemanticClosing content: copyright, links, author info▼
The <footer> typically contains copyright info, contact links, legal text, and site maps. Like <header>, a page can have multiple footers — each <article> or <section> can have its own. It does not have to be at the bottom visually.
Example
<footer>
  <p>&copy; 2024 My Company. All rights reserved.</p>
  <nav>
    <a href="/privacy">Privacy Policy</a>
    <a href="/terms">Terms of Service</a>
  </nav>
</footer>
<address>SemanticContact information for nearest article or body▼
The <address> element provides contact information for its nearest <article> or the entire document. It is for contact details — name, email, phone, physical address — not postal addresses in general. Rendered in italic by default.
Example
<footer>
  <address>
    Written by <a href="mailto:alex@example.com">Alex</a><br>
    123 Web Street, Internet City
  </address>
</footer>
<time>SemanticMachine-readable date and time▼
The <time> element represents a specific date, time, or duration in a machine-readable format. The datetime attribute provides the machine-readable value while the element's text content is human-friendly. Used by search engines and calendar apps.
Example
<time datetime="2024-03-15">March 15, 2024</time>
<time datetime="2024-03-15T14:30">2:30 PM</time>
<time datetime="P7D">7 days</time> <!-- duration -->
03

Text & Inline Elements

Tags that structure, format, and give meaning to text content.

<h1> — <h6>TextSix levels of headings — h1 is most important▼
Headings define the document outline. <h1> is the main page heading (use once per page for best SEO). h2–h6 create subheadings in hierarchical order. Never skip levels — go h1 → h2 → h3. Screen readers navigate pages by heading structure.
All six levels
<h1>Page Title (most important)</h1>
<h2>Chapter Heading</h2>
<h3>Section Subheading</h3>
<h4>Sub-section</h4>
<h5>Minor heading</h5>
<h6>Smallest heading (least important)</h6>
⚠ Only one <h1> per page. Never use headings just for their visual size — use CSS for that.
<p>TextParagraph — the most common block of text▼
The <p> element defines a paragraph of text. Browsers add space above and below paragraphs. Never use empty <p> tags for spacing — use CSS margin instead. Block-level: starts on a new line and takes full width.
Example
<p>This is the first paragraph of text.</p>
<p>This is the second paragraph. Browsers add
automatic spacing between paragraphs.</p>

<!-- Wrong: never use empty p for spacing -->
<p></p> <!-- BAD -->
<strong>TextStrong importance — rendered bold, meaningful emphasis▼
<strong> marks text of strong importance or urgency. Screen readers may stress the word. Visually rendered as bold by default. Use <strong> when the word matters semantically — not just for visual styling (use CSS font-weight: bold for that).
Example
<p><strong>Warning:</strong> Do not mix bleach and ammonia.</p>
<p>This item is <strong>out of stock</strong>.</p>
<em>TextEmphasis — rendered italic, changes sentence stress▼
<em> marks text with stress emphasis — it changes the meaning of the sentence. Screen readers will pronounce it with emphasis. Visually rendered italic. Different from <i> which is stylistic. "I never said that" vs "I never said that."
Example
<p>You <em>must</em> complete this step before continuing.</p>
<p>The <em>cat</em> sat on the mat. <!-- stresses "cat" --></p>
<code>TextInline code — uses monospace font▼
The <code> element marks a fragment of computer code — variable names, function names, file names, commands. Rendered in a monospace font. For multi-line code blocks, wrap <code> inside <pre>.
Example
<p>Use <code>console.log()</code> to debug JavaScript.</p>

<!-- Multi-line code block -->
<pre><code>
function greet(name) {
  return `Hello, ${name}!`;
}
</code></pre>
<pre>TextPreformatted text — preserves whitespace and line breaks▼
The <pre> element displays text exactly as written — preserving spaces, tabs, and line breaks. Uses monospace font. Used for code blocks, ASCII art, and text where formatting is meaningful. Always escape < and > inside <pre> as &lt; and &gt;.
Example
<pre>
  Name:    John Doe
  Email:   john@example.com
  Role:    Developer
</pre>
<blockquote>TextA long quotation from an external source▼
The <blockquote> marks a section quoted from another source. Use the cite attribute to link to the source URL. Browsers render it indented by default. For short inline quotes, use <q> instead.
Example
<blockquote cite="https://www.w3.org/TR/html52/">
  <p>The Web is for everyone.</p>
  <footer>— Tim Berners-Lee</footer>
</blockquote>
<br>TextLine break — inline, no closing tag needed▼
The <br> element creates a line break within text — useful for addresses, poetry, or anywhere whitespace is meaningful. It is a void element (no closing tag). Do not use <br> to create space between paragraphs — use CSS margin.
Example
<address>
  123 Main Street<br>
  Bangkok 10110<br>
  Thailand
</address>
⚠ Never use multiple <br> tags for spacing. Use CSS margin/padding instead.
<hr>TextThematic break — a horizontal rule/divider▼
The <hr> element represents a thematic break — a shift to a different topic. Visually rendered as a horizontal line. It is semantic (means a topic change), not just decorative. Use CSS border-top for purely decorative lines.
Example
<p>Chapter 1 ends here.</p>
<hr>
<p>Chapter 2 begins here.</p>
<mark>TextHighlighted text — like a yellow marker▼
The <mark> element highlights text for reference or relevance — like a highlighter pen. Default style is yellow background. Used for search result highlights, or to mark relevant parts of a quote. Different from <strong> — it marks relevance, not importance.
Example
<p>Search results for "HTML":
  Learn <mark>HTML</mark> in 30 days with our guide.
</p>
<small>TextSmall print — legal disclaimers, copyright, fine print▼
The <small> element represents small print — copyright notices, legal disclaimers, side comments. Renders text one size smaller. It is semantic — use CSS font-size for purely visual size reduction.
Example
<p>Buy now for $9.99!
  <small>Terms and conditions apply.</small>
</p>
<abbr>TextAbbreviation or acronym with tooltip▼
The <abbr> element marks an abbreviation or acronym. The title attribute provides the full expansion — shown as a tooltip on hover and read by screen readers. Always define abbreviations on first use.
Example
<p>The <abbr title="World Wide Web Consortium">W3C</abbr>
maintains the HTML standard.</p>

<p><abbr title="HyperText Markup Language">HTML</abbr>
is the language of the web.</p>
<sub> & <sup>TextSubscript and superscript text▼
<sub> renders text below the baseline (subscript) for chemical formulas and footnote markers. <sup> renders text above the baseline (superscript) for exponents, ordinals, and footnote numbers.
Example
<!-- Chemical formula -->
<p>Water is H<sub>2</sub>O</p>

<!-- Mathematical exponent -->
<p>E = mc<sup>2</sup></p>

<!-- Footnote reference -->
<p>According to Smith<sup>[1]</sup>, this is true.</p>
04

Lists

Structured ways to present collections of items — unordered, ordered, and definition lists.

<ul>ListUnordered list — bullet points, no sequence required▼
An unordered list shows items where order does not matter. Each item uses <li>. Default rendered as bullet points. Customizable via CSS list-style-type. Perfect for: navigation menus, ingredient lists, features, any set of items.
Example
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Nested list -->
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Backend</li>
</ul>
<ol>ListOrdered list — numbered steps, sequence matters▼
An ordered list shows items in sequential order — numbered by default. Use when the order of items matters: instructions, rankings, steps. The start attribute begins counting from a specific number. reversed counts down.
Key Attributes
startStarting number: <ol start="5"> begins at 5.
reversedBoolean: counts down instead of up.
type"1" (numbers), "A" (letters), "a", "I" (Roman), "i".
Example
<ol>
  <li>Preheat oven to 180°C</li>
  <li>Mix flour and sugar</li>
  <li>Add eggs and butter</li>
  <li>Bake for 30 minutes</li>
</ol>

<ol start="3" type="A">
  <li>This starts at "C"</li>
</ol>
<li>ListList item — used inside ul, ol, or menu▼
The <li> element is a list item — always a child of <ul>, <ol>, or <menu>. Can contain any content: text, links, images, even nested lists. The value attribute on <li> inside <ol> overrides the automatic number.
Example
<ol>
  <li>First item</li>
  <li value="5">This shows as "5."</li>
  <li>This shows as "6."</li>
</ol>
<dl> <dt> <dd>ListDefinition list — terms and descriptions▼
A definition list pairs terms (<dt>) with their descriptions (<dd>). Great for glossaries, FAQ sections, metadata key-value pairs, and dictionaries. One term can have multiple descriptions, and multiple terms can share one description.
Example
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language — the structure of web pages.</dd>

  <dt>CSS</dt>
  <dd>Cascading Style Sheets — the styling of web pages.</dd>

  <dt>JavaScript</dt>
  <dd>The programming language of the web.</dd>
</dl>
05

Links & Navigation

Hyperlinks are the core of the web — connecting pages, resources, and sections.

<a>LinkAnchor — the hyperlink tag connecting pages and resources▼
The <a> (anchor) element creates hyperlinks. The href attribute specifies the destination. Links can go to other pages, sections of the same page (#id), email addresses (mailto:), phone numbers (tel:), or files to download.
Key Attributes
hrefThe URL or destination. Required for a clickable link.
target"_blank" opens in new tab. "_self" (default) opens in same tab.
rel"noopener noreferrer" — required with target="_blank" for security.
downloadBoolean or filename — triggers file download instead of navigation.
titleTooltip text shown on hover.
All link types
<!-- External link -->
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  Visit Example
</a>

<!-- Internal page link -->
<a href="/about">About Us</a>

<!-- Jump to section on same page -->
<a href="#contact">Go to Contact</a>

<!-- Email link -->
<a href="mailto:hello@example.com">Email Us</a>

<!-- Phone link (great for mobile) -->
<a href="tel:+66912345678">Call Us</a>

<!-- Download file -->
<a href="/files/report.pdf" download="Annual-Report.pdf">
  Download Report
</a>
⚠ Always add rel="noopener noreferrer" to target="_blank" links to prevent security vulnerabilities.
id="" (anchor target)LinkCreates a named location on the page to link to▼
Any element with an id attribute becomes a linkable anchor. Link to it with href="#id-name". Essential for table of contents, "skip to content" accessibility links, and page sections. IDs must be unique per page.
Example
<!-- Navigation -->
<nav>
  <a href="#services">Services</a>
  <a href="#contact">Contact</a>
</nav>

<!-- Target sections -->
<section id="services">
  <h2>Our Services</h2>
</section>

<section id="contact">
  <h2>Contact Us</h2>
</section>
06

Media

Images, video, audio, and embedded content — making web pages rich and visual.

<img>MediaEmbeds an image — void element, no closing tag▼
The <img> element embeds an image. The alt attribute is essential — it describes the image for screen readers and displays when the image fails to load. Never leave alt empty for meaningful images (leave empty for decorative ones).
Key Attributes
srcURL of the image. Required.
altAlternative text description. Required for accessibility.
width / heightSet both to prevent layout shift during loading.
loading"lazy" defers off-screen images. "eager" loads immediately.
srcsetMultiple image sources for different screen sizes/resolutions.
Example
<!-- Basic image -->
<img src="photo.jpg" alt="A mountain at sunset" width="800" height="600">

<!-- Lazy loading -->
<img src="photo.jpg" alt="Description" loading="lazy">

<!-- Responsive srcset -->
<img
  src="small.jpg"
  srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1200w"
  sizes="(max-width: 480px) 100vw, 800px"
  alt="Responsive image">

<!-- Decorative image (skip screen readers) -->
<img src="divider.svg" alt="" role="presentation">
<figure> & <figcaption>MediaSelf-contained media with an optional caption▼
<figure> wraps self-contained content referenced from the main flow — images, diagrams, code, charts. <figcaption> provides a visible caption. The figure can be moved without affecting the main document.
Example
<figure>
  <img src="chart.png" alt="Sales data 2024">
  <figcaption>Figure 1: Sales increased 32% in Q4 2024.</figcaption>
</figure>
<video>MediaEmbeds a video with browser-native controls▼
The <video> element embeds video files. Use multiple <source> children for different formats. Always include controls or provide custom JavaScript controls. The fallback text is shown in browsers that don't support the tag.
Key Attributes
srcVideo file URL.
controlsBoolean — shows play/pause/volume controls.
autoplayPlays automatically (use with muted to work in Chrome).
mutedStarts muted — required for autoplay in modern browsers.
loopReplays automatically when finished.
posterImage to show before the video plays.
Example
<video controls width="640" poster="thumbnail.jpg">
  <source src="video.webm" type="video/webm">
  <source src="video.mp4"  type="video/mp4">
  <p>Your browser does not support HTML video.</p>
</video>

<!-- Background video (no controls, autoplay) -->
<video autoplay muted loop playsinline>
  <source src="bg.mp4" type="video/mp4">
</video>
<audio>MediaEmbeds audio with browser-native controls▼
The <audio> element embeds sound files. Same pattern as <video> — use <source> children for format fallbacks. Provide controls so users can interact with the audio.
Example
<audio controls>
  <source src="podcast.ogg" type="audio/ogg">
  <source src="podcast.mp3" type="audio/mpeg">
  <p>Your browser does not support HTML audio.</p>
</audio>
<iframe>MediaEmbeds another HTML page or external content▼
The <iframe> (inline frame) embeds another web page inside the current page. Commonly used for Google Maps, YouTube embeds, payment widgets, and third-party tools. Use sandbox for security when embedding untrusted content.
Example
<!-- YouTube embed -->
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  width="560"
  height="315"
  title="YouTube video player"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media"
  allowfullscreen>
</iframe>

<!-- Google Maps -->
<iframe
  src="https://maps.google.com/maps?q=Bangkok&output=embed"
  width="600" height="400"
  loading="lazy">
</iframe>
⚠ Always add title attribute to iframes for screen reader users.
07

Tables

Displaying data in rows and columns — with proper semantic structure for accessibility.

<table> — Full StructureTableAll table tags — table, thead, tbody, tfoot, tr, th, td, caption▼
An HTML table organizes data into rows and columns. Use the full semantic structure for accessibility: <caption> (title), <thead> (header rows), <tbody> (data rows), <tfoot> (summary row), <tr> (table row), <th> (header cell — bold, centered), <td> (data cell).
Key Attributes for td/th
colspanSpan multiple columns: colspan="2".
rowspanSpan multiple rows: rowspan="3".
scopeFor <th>: "col" or "row" — tells screen readers what the header labels.
Full accessible table
<table>
  <caption>Monthly Sales Report</caption>
  <thead>
    <tr>
      <th scope="col">Month</th>
      <th scope="col">Revenue</th>
      <th scope="col">Units</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>January</td>
      <td>$12,500</td>
      <td>250</td>
    </tr>
    <tr>
      <td>February</td>
      <td>$15,200</td>
      <td>304</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th scope="row">Total</th>
      <td>$27,700</td>
      <td>554</td>
    </tr>
  </tfoot>
</table>
Colspan & Rowspan
<table>
  <tr>
    <th colspan="2">Name</th> <!-- spans 2 columns -->
    <th>Age</th>
  </tr>
  <tr>
    <td>John</td>
    <td>Doe</td>
    <td rowspan="2">30</td> <!-- spans 2 rows -->
  </tr>
</table>
08

Forms & Input

Collecting user data — text, choices, files, and actions.

<form>FormContainer for user input elements with a submission action▼
The <form> element wraps input elements and defines how to submit data. The action specifies where to send data, method specifies how (GET adds to URL, POST sends in body). Forms can also be handled entirely by JavaScript.
Key Attributes
actionURL to submit form data to. Omit for JavaScript-only forms.
method"get" (data in URL) or "post" (data in body, more secure).
enctype"multipart/form-data" — required for file uploads.
novalidateDisables browser's built-in validation (for custom validation).
Contact form
<form action="/submit" method="post">
  <label for="name">Name</label>
  <input type="text" id="name" name="name" required>

  <label for="email">Email</label>
  <input type="email" id="email" name="email" required>

  <button type="submit">Send Message</button>
</form>
<input>FormThe most versatile form element — 20+ types available▼
The <input> element creates interactive controls. Its behavior is controlled by the type attribute. It is a void element (no closing tag). Always pair with a <label> for accessibility — use the for/id connection.
All major input types
<!-- Text inputs -->
<input type="text"     placeholder="Enter text">
<input type="email"    placeholder="you@example.com">
<input type="password" placeholder="Password">
<input type="tel"      placeholder="+66 12 345 6789">
<input type="url"      placeholder="https://example.com">
<input type="search"   placeholder="Search...">

<!-- Number inputs -->
<input type="number" min="0" max="100" step="5">
<input type="range"  min="0" max="10" value="5">

<!-- Date/time inputs -->
<input type="date">
<input type="time">
<input type="datetime-local">
<input type="month">
<input type="week">

<!-- Choice inputs -->
<input type="checkbox" id="agree" name="agree">
<label for="agree">I agree to terms</label>

<input type="radio" name="size" value="s"> Small
<input type="radio" name="size" value="m"> Medium

<!-- Misc -->
<input type="color"  value="#ff0000">
<input type="file"   accept=".pdf,.jpg,.png">
<input type="hidden" name="token" value="abc123">
<input type="submit" value="Submit Form">
<label>FormAssociates a text label with a form control▼
The <label> element links descriptive text to a form control. Clicking the label focuses or activates its control. Always use labels — they are critical for accessibility. Connect via the for attribute matching the input's id.
Example
<!-- Method 1: for/id connection -->
<label for="email">Email address</label>
<input type="email" id="email" name="email">

<!-- Method 2: wrap the input -->
<label>
  Username
  <input type="text" name="username">
</label>
<textarea>FormMulti-line text input — user-resizable by default▼
The <textarea> element creates a multi-line text field. Unlike <input>, it has both opening and closing tags. Set rows and cols for default size, or use CSS. Content between tags is the default value.
Example
<label for="message">Your Message</label>
<textarea
  id="message"
  name="message"
  rows="5"
  cols="40"
  placeholder="Write your message here..."
  maxlength="500"
></textarea>
<select> & <option>FormDropdown list of selectable options▼
The <select> creates a dropdown menu. Each choice is an <option>. Group related options with <optgroup>. The multiple attribute lets users pick several options (hold Ctrl/Cmd to select).
Example
<label for="country">Country</label>
<select id="country" name="country">
  <option value="">-- Select country --</option>
  <optgroup label="Asia">
    <option value="th">Thailand</option>
    <option value="jp">Japan</option>
    <option value="sg" selected>Singapore</option>
  </optgroup>
  <optgroup label="Europe">
    <option value="fr">France</option>
    <option value="de">Germany</option>
  </optgroup>
</select>
<button>FormClickable button — can contain HTML, three types▼
The <button> element creates a clickable button that can contain HTML content (icons, images). Three types: submit (sends form, default), reset (clears form), button (does nothing without JavaScript). Prefer <button> over <input type="submit">.
Example
<!-- Form submit -->
<button type="submit">Submit Form</button>

<!-- Reset form -->
<button type="reset">Clear All</button>

<!-- JavaScript trigger -->
<button type="button" onclick="handleClick()">
  Click Me
</button>

<!-- Disabled button -->
<button type="submit" disabled>Please fill the form</button>
<fieldset> & <legend>FormGroups related form controls with a labelled border▼
<fieldset> groups related inputs together (visually with a border). <legend> provides the caption that appears in the border. Essential for grouping radio buttons or checkboxes that form a single logical choice. Screen readers announce the legend before each input in the group.
Example
<fieldset>
  <legend>Preferred contact method</legend>
  <label><input type="radio" name="contact" value="email"> Email</label>
  <label><input type="radio" name="contact" value="phone"> Phone</label>
  <label><input type="radio" name="contact" value="sms"> SMS</label>
</fieldset>

Meta & Head Tags

Invisible but essential — telling browsers, search engines, and social platforms about your page.

<meta charset>MetaCharacter encoding — always UTF-8▼
The charset meta tag declares the character encoding of the HTML document. Always use UTF-8 — it supports every character in every language including Thai, Chinese, Arabic, emoji, and special symbols. Must appear before <title>.
Example
<head>
  <!-- Must be first in <head> -->
  <meta charset="UTF-8">
  <title>My Page</title>
</head>
<meta name="viewport">MetaControls how the page scales on mobile devices▼
The viewport meta tag is essential for responsive design. Without it, mobile browsers render the page at desktop width (typically 960px) and then scale it down — making text tiny. This tag tells mobile browsers to match the device width.
Example
<!-- Standard responsive setup -->
<meta name="viewport" content="width=device-width, initial-scale=1">

<!-- Prevent zoom (accessibility concern!) -->
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">
⚠ Avoid maximum-scale=1 — it prevents users from zooming in, which is an accessibility violation.
<meta name="description">MetaPage description shown in search engine results▼
The description meta tag provides a summary of the page shown in search engine results beneath the title. While it doesn't directly affect ranking, a compelling description improves click-through rates. Keep under 160 characters.
SEO meta tags
<meta name="description" content="Learn HTML with 87 tags, live examples, and clear explanations. The complete developer reference.">

<meta name="author"      content="Jane Smith">
<meta name="keywords"    content="HTML, reference, tags">
<meta name="robots"      content="index, follow">
<meta http-equiv="refresh" content="30"> <!-- auto-refresh after 30s -->
Open Graph <meta>MetaControls how links appear when shared on social media▼
Open Graph tags (created by Facebook) control what appears when your page is shared on Facebook, Twitter/X, LinkedIn, and WhatsApp — title, description, and image. Twitter has its own additional tags.
Social sharing tags
<!-- Open Graph (Facebook, LinkedIn, WhatsApp) -->
<meta property="og:title"       content="Book HTML Reference">
<meta property="og:description"  content="87 HTML tags with examples">
<meta property="og:image"        content="https://example.com/preview.jpg">
<meta property="og:url"          content="https://example.com">
<meta property="og:type"         content="website">

<!-- Twitter Card -->
<meta name="twitter:card"        content="summary_large_image">
<meta name="twitter:title"       content="Book HTML Reference">
<meta name="twitter:description"  content="87 HTML tags with examples">
<meta name="twitter:image"       content="https://example.com/preview.jpg">
<link>MetaLinks external resources: CSS, fonts, icons, manifests▼
The <link> element in <head> connects external resources to the document. Most commonly used for CSS stylesheets, favicons, and web fonts. The rel attribute describes the relationship.
Example
<!-- CSS stylesheet -->
<link rel="stylesheet" href="styles.css">

<!-- Favicon -->
<link rel="icon"       href="favicon.ico">
<link rel="icon"       type="image/svg+xml" href="icon.svg">
<link rel="apple-touch-icon" href="apple-icon.png">

<!-- Google Fonts -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter">

<!-- Canonical URL (prevents duplicate content SEO issues) -->
<link rel="canonical" href="https://example.com/page">

<!-- Web App Manifest -->
<link rel="manifest" href="/manifest.json">

Scripts & Style

Embedding code and styles directly in HTML — the bridge between structure and behavior.

<script>ScriptEmbeds or links JavaScript — defer or place before </body>▼
The <script> element embeds or links JavaScript. Inline scripts block HTML parsing. External scripts with defer execute after HTML is parsed. async loads in parallel but executes as soon as loaded (order not guaranteed). Place most scripts at the end of <body> or use defer.
Key Attributes
srcURL of external JavaScript file.
deferExecute after HTML is fully parsed. Preserves order.
asyncLoad in parallel, execute immediately. Order not guaranteed.
type="module"Enable ES modules (import/export). Deferred by default.
Example
<!-- Inline JavaScript -->
<script>
  console.log('Hello from inline script!');
  document.getElementById('btn').addEventListener('click', () => {
    alert('Clicked!');
  });
</script>

<!-- External file with defer (recommended) -->
<script src="app.js" defer></script>

<!-- ES Module -->
<script type="module">
  import { greet } from './utils.js';
  greet('World');
</script>

<!-- Async (analytics, ads) -->
<script src="analytics.js" async></script>
<style>StyleEmbeds CSS directly in the HTML document▼
The <style> element embeds CSS directly in the HTML. Goes in <head>. Useful for critical CSS (above-the-fold styles loaded immediately) or page-specific styles. For larger projects, prefer external <link rel="stylesheet"> for caching and reuse.
Example
<head>
  <style>
    body {
      font-family: Arial, sans-serif;
      margin: 0;
      background: #f5f5f5;
    }

    h1 {
      color: #2563eb;
      font-size: 2rem;
    }

    .container {
      max-width: 1200px;
      margin: 0 auto;
      padding: 0 20px;
    }
  </style>
</head>
<noscript>ScriptFallback content shown when JavaScript is disabled▼
The <noscript> element provides fallback content for users who have JavaScript disabled or whose browser doesn't support it. In <head>, it can contain <link>, <style>, or <meta> tags. In <body>, it can contain any HTML.
Example
<noscript>
  <p>This site requires JavaScript to work properly.
  Please <a href="https://enable-javascript.com">enable JavaScript</a>.</p>
</noscript>

<!-- In <head> for analytics fallback -->
<noscript>
  <img src="tracking.gif" alt="" width="1" height="1">
</noscript>
<template>ScriptHTML that is not rendered — cloned and used by JavaScript▼
The <template> element holds HTML that is not rendered when the page loads. Its content is inert (scripts don't run, images don't load) until cloned by JavaScript. Used for client-side templating — dynamically creating repeated HTML structures.
Example
<!-- Define the template -->
<template id="card-template">
  <div class="card">
    <h3 class="title"></h3>
    <p  class="desc"></p>
  </div>
</template>

<!-- Use with JavaScript -->
<script>
  const tmpl  = document.getElementById('card-template');
  const clone = tmpl.content.cloneNode(true);
  clone.querySelector('.title').textContent = 'HTML';
  clone.querySelector('.desc').textContent  = 'Markup language.';
  document.body.appendChild(clone);
</script>
<canvas>ScriptA drawing surface controlled entirely by JavaScript▼
The <canvas> element provides a pixel-based drawing area. By itself it is blank — all drawing is done with the Canvas 2D API via JavaScript. Used for games, data visualizations, image manipulation, and animations. For complex charts, use libraries like Chart.js.
Example
<canvas id="myCanvas" width="400" height="200">
  Canvas not supported in your browser.
</canvas>

<script>
  const canvas = document.getElementById('myCanvas');
  const ctx    = canvas.getContext('2d');

  // Draw a blue rectangle
  ctx.fillStyle = '#2563eb';
  ctx.fillRect(10, 10, 150, 100);

  // Draw text
  ctx.fillStyle = 'white';
  ctx.font = '20px Arial';
  ctx.fillText('Hello Canvas!', 20, 60);

  // Draw a circle
  ctx.beginPath();
  ctx.arc(300, 100, 60, 0, Math.PI * 2);
  ctx.fillStyle = '#f59e0b';
  ctx.fill();
</script>
